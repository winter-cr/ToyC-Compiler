# ToyC 编译系统实践报告

## 1. 项目概述

本项目实现了一个 ToyC 编译器，输入为 ToyC 源程序，输出为 RV32 汇编代码。编译器采用经典多阶段结构：词法分析、语法分析、AST、语义分析、中间表示、优化和目标代码生成。

评测入口支持两种常见构建方式：

```bash
cmake -S . -B build && cmake --build build
make
```

编译器从 `stdin` 读取源程序，向 `stdout` 输出汇编，支持 `-opt` 开启优化。

## 2. 目录结构

```text
src/                    前端和 AST 到 IR 降低
backend/include/        后端公共接口
backend/src/            后端 IR、优化器、RV32 代码生成
tests/                  功能、语义、控制流和性能样例
docs/                   实践报告与设计文档
scripts/                测试脚本
```

该结构与常见编译器项目分层一致：前端负责语言识别与语义检查，后端负责 IR 优化和目标代码生成，构建入口位于仓库根目录。

## 3. 前端设计

词法分析器识别关键字、标识符、整数、运算符和分隔符。减号始终作为运算符处理，避免将 `x-y` 误判为非法标识符。

语法分析器构建 AST，支持函数定义、变量/常量声明、表达式、条件语句、循环、`break`、`continue` 和 `return`。

语义分析器维护符号表和作用域，检查未声明标识符、重复定义、常量重赋值、函数参数数量、`void` 函数表达式使用、循环外 `break/continue`、除零和返回路径。返回路径分析采用递归语句判断，能识别完整 `if/else` 和常量真循环中的确定返回。

## 4. 中间表示与后端

后端 IR 是显式控制流三地址码，核心指令包括 `Copy`、`Unary`、`Binary`、`Label`、`Jump`、`Branch`、`Call` 和 `Return`。

RV32 代码生成遵循简化 ILP32 调用约定：前 8 个参数使用 `a0..a7`，更多参数走调用者栈空间，返回值使用 `a0`，函数保存并恢复 `ra`、`s0` 和实际使用的 `s` 寄存器。

## 5. 性能优化

本轮优化重点包括：

- 将 IR 死临时值删除改为一次反向活跃性扫描，降低优化阶段复杂度。
- 将局部变量、函数参数和虚拟临时值统一纳入 `s1..s11` 分配，减少循环内 `lw/sw`。
- 对 `Copy`、`Return`、`Branch` 增加寄存器快速路径，减少 `t0` 中转指令。
- 默认 CMake 构建类型设为 Release，Makefile 使用 `-O2` 构建编译器自身。
- 保留跳转穿透、死跳转移除、常量折叠、常量条件分支简化等已有优化。

这些优化对性能样例 `loop_sum.tc` 和 `fib_heavy.tc` 最直接：循环变量保留在寄存器中，递归参数和中间值减少栈访存。

## 6. 验证情况

已在本地使用 `g++` 手动构建验证：

- 主编译器构建通过。
- 后端单元测试通过。
- `tests/performance/loop_sum.tc` 和 `tests/performance/fib_heavy.tc` 在 `-opt` 下编译通过。
- `tests/control_flow/*.tc` 批量编译通过。
- 针对 `x-y` 表达式和 `while (1) { return 0; }` 返回路径的回归样例通过。

本机环境缺少 `cmake` 命令，因此 CMake 构建未在本机执行，但项目根目录保留了完整 `CMakeLists.txt`，并补充了通用 `Makefile`。
