## 三、修复效果评估

| 修复项 | Issue | 效果 | 影响测试 |
|--------|:---:|:---:|------|
| Block save/restore locals_ | #13 | ✅ 有效 | f07, f13, f25 通过 |
| assignRegisters 返回空 | #12 | ⚠️ 部分有效 | p09 不再超时，但性能分无明显提升 |
| ExprStmt→FuncCall 路由 | #14 | ❌ 未解决 | f10, f24 仍 crash |
| 全局变量 null 检查 | #15 | ❌ 未解决 | f21 仍 crash |

### 分析

1. **Block 作用域修复是唯一完全生效的改动**，说明 AstToIr 中 `locals_` 的作用域隔离对变量遮蔽是必要且正确的

2. **void 函数 crash 根因不在 ExprStmt 路由**：crash 可能发生在 void 函数定义阶段（FuncDef visitor）或代码生成阶段（`ret` 无值处理），需要更深入的诊断

3. **性能修复效果有限**：完全去掉寄存器分配使 p09 从超时变通过，但纯栈访存方案比寄存器 spill 链并没有显著性能优势。需要一个折中方案（如只分配前 2-3 个最常用的 temp 到寄存器）

4. **f16/f18 需要更多信息**：没有 OJ 测试用例很难精确定位
